## 7.  整数反转

**题目：**

```
给出一个 32 位的有符号整数，你需要将这个整数中每位上的数字进行反转。
```

**示例：**

```
1.
输入: 123
输出: 321
2.
输入: -123
输出: -321
3.
输入: 120
输出: 21

注意:
假设我们的环境只能存储得下32位的有符号整数，则其数值范围为 [−2^31,  2^31 − 1]。请根据这个假设，如果反转后整数溢出那么就返回 0。
```

**思路：**

刚看到这道题的时候，容易想到将整数转成字符串处理，对字符串进行反转，但是这样需要考虑的情况比较多，比如负号的处理，和超出数值范围的异常处理，反而将简单的问题复杂化。可以用简单的数学思维去解答，从低位到高位依次遍历取余，然后将余数遍历乘`10`进位反转，最后将结果强转成`int`类型，如果超出了数据范围，数据会取补码，如此也解决了取值范围的问题。

**代码：**

```java
    public int reverse(int x) {
        long n = 0;
        while (x != 0) {
            n = n * 10 + x % 10;
            x = x / 10;
        }
        return (int) n == n ? (int) n : 0;
    }
```
